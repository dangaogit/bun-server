## 如何使用

1. **遇到问题时**：先在对应分类下查找是否有类似问题
2. **解决问题后**：记录问题、原因和解决方案到对应分类
3. **持续更新**：随着框架演进，及时更新文档

## 贡献指南

记录新问题时，请包含：

1. **问题描述**：清晰描述错误现象
2. **错误信息**：完整的错误堆栈或日志
3. **问题原因**：根本原因分析
4. **解决方案**：详细的修复步骤
5. **相关代码**：问题代码和修复后的代码对比
6. **关键点**：总结要点，方便快速理解
